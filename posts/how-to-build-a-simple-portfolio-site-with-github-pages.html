Building a Simple Portfolio Site with GitHub Pages

Productivity • Updated: 2026-03-04

TL;DR

  • GitHub Pages is a free hosting service for static websites.
  • You’ll need a GitHub account and basic knowledge of HTML and Markdown.
  • This guide walks you through the creation of a simple portfolio site.
  • A well-structured portfolio can enhance job prospects and professional presence.
  • Common pitfalls include neglecting mobile responsiveness and not updating content regularly.

Who this is for

This guide is ideal for web developers, graphic designers, photographers, and anyone looking to showcase their work online. If you’re a freelancer, job seeker, or simply want to demonstrate your skills, a portfolio site can make a significant impact. Knowledge of HTML is helpful, but you don't need to be a coding expert to get started.

Key ideas

Creating a portfolio site on GitHub Pages allows for easy updates and version control, thanks to Git. The platform is integrated with GitHub, making it simple to host your site alongside your project repositories. You can use templates or build from scratch and utilize Markdown for content formatting.

Step by step

1. Set Up Your GitHub Account: If you don't have a GitHub account yet, sign up at github.com. After verifying your email, you’ll gain access to creating repositories.

2. Create a New Repository: Click on 'New' in the repositories section. Name your repository in the format `.github.io`. Choose 'Public' and click on 'Create repository.'

3. Choose a Template: You can start with a Jekyll template for easier management. Visit GitHub Jekyll themes to find one that suits your style.

4. Clone the Repository: Use Git to clone your repository to your local machine. Open your terminal and execute: git clone https://github.com//.github.io.git.

5. Add Your Content: Inside the cloned folder, edit the `index.html` or `_config.yml` file. Update it with your personal information, project links, and images. Use Markdown for easy formatting.

6. Push Your Changes: After making updates, save your files. Use commands to stage, commit, and push your changes: git add ., git commit -m 'Your message', git push.

7. Enable GitHub Pages: Go to the 'Settings' tab in your repository, scroll down to the “GitHub Pages” section, choose the main branch or `/docs` folder, and save. Your site will be live in a moment!

Comparisons

When comparing GitHub Pages to other hosting options:

  • Cost: GitHub Pages is free for public repositories, whereas platforms like Squarespace and Wix require monthly payments.
  • Ease of Use: WordPress offers more customization but has a steeper learning curve than GitHub Pages' streamlined setup.
  • Control: GitHub Pages provides greater version control since it's built on Git, allowing for easy rollbacks on changes.
  • SEO: Both options can be optimized for SEO, but GitHub Pages allows for direct editing of code for advanced optimization.

Common mistakes

One common mistake is neglecting to optimize for mobile devices. Check your site on various screen sizes to ensure responsiveness. Another issue can be not utilizing Git effectively—remember to commit changes regularly to keep track of your updates. Lastly, be careful with file paths for images and CSS; using the wrong path can result in broken links.

Checklist

  • Sign up for GitHub and create a repository.
  • Choose an appropriate template or create from scratch.
  • Edit content responsibly, using clear headings and proper formatting.
  • Test your site across different browsers and devices.
  • Keep your portfolio updated with new projects.
  • Ensure proper SEO practices are followed.
